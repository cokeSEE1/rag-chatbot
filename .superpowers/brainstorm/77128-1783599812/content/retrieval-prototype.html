RAG（检索增强生成）是一种结合信息检索和文本生成的 AI 架构。

核心区别：

  • 普通 AI 问答仅依赖模型训练时记忆的知识，存在知识截止日期和幻觉风险
  • RAG 在回答前先从外部知识库检索相关文档，将检索结果作为上下文提供给 LLM

工作流程：提问 → 向量检索 → 召回 Top-K → 拼接 Prompt → 生成回答

优势：知识可实时更新、回答可溯源、减少幻觉、支持私有知识库。

RAG 和微调哪个更好？

#1 88% RAG 和微调不是互斥的，而是互补的。RAG 适合需要实时更新知识的场景，微调适合需要改变模型行为或风格的场景。通常建议先尝试 RAG，效果不够再考虑微调。
#2 76% 成本对比：RAG 只需支付 Embedding + 检索的算力成本；微调需要准备高质量训练数据 + GPU 训练时间。对于大多数企业应用，RAG 是更经济的首选方案。
#3 71% 决策框架：知识更新频率高 → RAG；需要特定格式输出 → 微调；两者都需要 → RAG + 微调组合。从 Prompt → RAG → Few-shot → Fine-tuning 逐级升级。

RAG 和微调不是二选一，而是互补关系：

  • RAG 适合知识频繁更新的场景，成本低、见效快
  • 微调 适合需要改变模型行为/输出格式的场景

选型建议：Prompt → RAG → Few-shot → Fine-tuning 逐级升级。大多数场景 RAG 就够用。
