什么是 RAG？它和普通 AI 问答有什么区别？
检索完成 5 个片段 · 0.3s
1 RAG（Retrieval-Augmented Generation）是一种将信息检索与文本生成相结合的 AI 架构，核心思想是在 LLM 生成回答之前，先从外部知识库中检索相关信息作为上下文参考。
2 传统 LLM 问答依赖模型训练时记忆的知识，存在截止日期限制和幻觉风险。RAG 通过实时检索外部文档来解决这些问题，使回答更加准确、可溯源。
3 RAG 典型工作流程：用户提问 → Embedding 向量化 → 向量相似度检索 → 召回 Top-K 文档 → 拼接 Prompt → LLM 生成回答 → 返回结果。
4 RAG 主要优势：知识可实时更新无需重新训练、回答可溯源到具体文档、大幅减少幻觉、支持私有知识库问答。
5 相比传统 FAQ 问答系统，RAG 不需人工编写问答对，上传文档即可自动完成清洗、分块、向量化、存储和检索全过程。

RAG（检索增强生成）是一种结合信息检索和文本生成的 AI 架构。

与普通 AI 问答的核心区别：

  • 普通 AI 仅依赖模型记忆的知识，存在知识截止和幻觉问题
  • RAG 先从外部知识库检索相关文档，再基于检索结果生成回答

工作流程：提问 → 向量检索 → 召回 Top-K → Prompt → 生成

优势：知识实时更新、回答可溯源、减少幻觉、支持私有知识库。

RAG 和微调哪个更好？
检索完成 3 个片段 · 0.2s
1 RAG 和微调不是互斥的，而是互补的。RAG 适合需实时更新知识的场景，微调适合需改变模型行为或输出风格的场景。建议先尝试 RAG。
2 成本对比：RAG 只需 Embedding + 检索算力；微调需要高质量训练数据 + GPU 训练时间。对大多数企业应用，RAG 是更经济的首选方案。
3 决策框架：知识更新频率高 → RAG；需特定输出格式 → 微调；两者都需要 → RAG + 微调组合。按 Prompt → RAG → Few-shot → Fine-tuning 逐级升级。

RAG 和微调不是二选一，而是互补关系：

  • RAG — 知识频繁更新，成本低见效快
  • 微调 — 改变模型行为或输出格式

选型建议：Prompt → RAG → Few-shot → Fine-tuning 逐级升级，大多数场景 RAG 就够。
